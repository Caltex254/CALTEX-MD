    64→## Overview
    65→
    66→CALTEX MD is a full-featured WhatsApp multi-device bot built on the Baileys library. It provides a rich command system, multi-provider AI integration, a plugin architecture, group moderation tools, media handling, scheduled messaging, and a beautiful Next.js web dashboard for complete bot management.
    67→
    68→## Key Features
    69→
    70→### WhatsApp Integration
    71→- Multi-device support via Baileys MD
    72→- QR code authentication
    73→- **Pairing code authentication** — link via phone number without scanning
    74→- Auto-reconnect with configurable retry logic
    75→- Multi-session management
    76→- Message send/receive with full media support
    77→
    78→### AI Integration
    79→- Multi-provider support: **OpenAI**, **Gemini**, **Claude**, **Ollama**, **Custom**
    80→- Conversation memory with TTL-based cleanup
    81→- Chat, image generation, code generation, translation, summarization, rewriting, and explanation
    82→- Configurable system prompts and model parameters
    83→
    84→### Plugin System
    85→- Dynamic plugin loading and unloading
    86→- Custom command registration via plugins
    87→- Plugin enable/disable from the dashboard or WhatsApp commands
    88→- Hot-reload support for development
    89→- Install plugins from file paths
    90→- Full lifecycle management: install, remove, enable, disable, reload
    91→
    92→### Web Dashboard
    93→- Real-time bot status and health monitoring
    94→- QR code scanning interface
    95→- **Pairing code panel** — link via phone number
    96→- Session management (create, delete, export, import)
    97→- Group management and moderation controls
    98→- AI configuration panel
    99→- Scheduled message management

   141→## Quick Start
   142→
   143→### Prerequisites
   144→
   145→- **Node.js** >= 20 or **Bun** >= 1.0
   146→- **npm** or **bun** package manager
   147→- A WhatsApp account with an active phone number
   148→- (Optional) AI provider API keys for AI features
   149→
   150→### Installation
   151→
   152→```bash
   153→# Clone the repository
   154→git clone https://github.com/caltex-md/caltex-md.git
   155→cd caltex-md
   156→
   157→# Install dependencies
   158→npm install
   159→# or with bun
   160→bun install
   161→
   162→# Set up environment variables
   163→cp .env.example .env
   164→# Edit .env with your configuration
   165→
   166→# Initialize the database
   167→npx prisma db push
   168→npx prisma generate
   169→
   170→# Build the dashboard
   171→npm run build
   172→```
   173→
   174→### Running the Bot
   175→
   176→```bash
   177→# Start all services (Dashboard + Bot + WebSocket)
   178→npm start
   179→
   180→# Development mode (Dashboard only)
   181→npm run dev
   182→
   183→# Start bot service separately
   184→cd mini-services/caltex-bot
   185→bun run dev
   186→```
   187→
   188→When the bot starts, it will display a QR code in the console. Scan it with WhatsApp:
   189→
   190→> **WhatsApp** > **Settings** > **Linked Devices** > **Link a Device**
   191→
   192→### Using Pairing Code (New in v1.1.0)
   193→
   194→Instead of scanning a QR code, you can link your bot using a phone number:
   195→
   196→1. Open the **Dashboard** > **Pairing Code** panel
   197→2. Enter your WhatsApp phone number (e.g., `6281234567890`)
   198→3. Click **Request Pairing Code**
   199→4. Enter the received code on your phone:
   200→   > **WhatsApp** > **Settings** > **Linked Devices** > **Link with Phone Number**
   201→
   202→You can also request a pairing code via the API:
   203→
   204→```bash
   205→curl -X POST http://localhost:3000/api/pairing-code \
   206→  -H "Authorization: Bearer <your-token>" \
   207→  -H "Content-Type: application/json" \
   208→  -d '{"phoneNumber":"6281234567890"}'
   209→```
   210→
   211→### Docker Quick Start
   212→
   213→```bash
   214→# Start all services with Docker Compose
   215→docker compose up -d
   216→
   217→# View logs
   218→docker compose logs -f
   219→
   220→# Stop all services
   221→docker compose down
   222→```

   226→## Configuration
   227→
   228→All configuration is managed through environment variables in the `.env` file:
   229→
   230→| Variable | Description | Default |
   231→|---|---|---|
   232→| `NODE_ENV` | Runtime environment | `development` |
   233→| `PORT` | Dashboard port | `3000` |
   234→| `DATABASE_URL` | SQLite database path | `file:./data/caltex.db` |
   235→| `BOT_API_URL` | Bot service URL | `http://localhost:3031` |
   236→| `WS_URL` | WebSocket service URL | `http://localhost:3003` |
   237→| `NEXTAUTH_SECRET` | NextAuth secret key | `change-me-in-production` |
   238→| `NEXTAUTH_URL` | NextAuth callback URL | `http://localhost:3000` |
   239→| `JWT_SECRET` | JWT signing secret | `caltex-md-bot-secret-key-2024` |
   240→| `ADMIN_USER` | Dashboard admin username | `admin` |
   241→| `ADMIN_PASS` | Dashboard admin password | `admin123` |
   242→| `PASSWORD_SALT` | Password hashing salt | `caltex-salt` |
   243→
   244→Bot-specific configuration is also managed via the `bot-config.json` and `ai-config.json` files, which can be edited from the dashboard.

   706→## Plugin Development
   707→
   708→CALTEX MD supports a plugin architecture for extending bot functionality.
   709→
   710→### Plugin Structure
   711→
   712→```typescript
   713→import type { Plugin, CommandContext } from '@/lib/plugin-loader';
   714→
   715→const myPlugin: Plugin = {
   716→  name: 'my-plugin',
   717→  version: '1.0.0',
   718→  description: 'My custom plugin',
   719→  author: 'Your Name',
   720→  enabled: true,
   721→  commands: [
   722→    {
   723→      name: 'hello',
   724→      description: 'Say hello',
   725→      category: 'custom',
   726→      aliases: ['hi', 'hey'],
   727→      cooldown: 3,
   728→      isPremiumOnly: false,
   729→      handler: async (ctx: CommandContext) => {
   730→        await ctx.reply('Hello from my plugin! 👋');
   731→      },
   732→    },
   733→  ],
   734→};
   735→
   736→export default myPlugin;
   737→```
   738→
   739→### Command Context
   740→
   741→Each command handler receives a `CommandContext` object with:
   742→
   743→| Property | Type | Description |
   744→|---|---|---|
   745→| `sock` | `WASocket` | WhatsApp socket instance |
   746→| `jid` | `string` | Chat JID |
   747→| `sender` | `string` | Sender JID |
   748→| `args` | `string[]` | Command arguments |
   749→| `text` | `string` | Full message text |
   750→| `isGroup` | `boolean` | Whether the message is from a group |
   751→| `isOwner` | `boolean` | Whether the sender is a bot owner |
   752→| `isAdmin` | `boolean` | Whether the sender is a group admin |
   753→| `reply(text)` | `function` | Reply to the message |
   754→| `react(emoji)` | `function` | React to the message |
   755→
   756→### Installing Plugins
   757→
   758→1. **Via WhatsApp command:** `!installplugin ./custom-plugins/my-plugin.ts`
   759→2. **Via dashboard:** Plugins panel > Install
   760→3. **Via file system:** Place your plugin file in `src/lib/commands/` and it will auto-load
   761→4. Enable/disable from the dashboard Plugins panel or via `!enableplugin`/`!disableplugin`
   762→5. Hot-reload during development with `!reloadplugin <name>`
