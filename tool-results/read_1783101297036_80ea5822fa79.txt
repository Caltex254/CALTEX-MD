    67→## Overview
    68→
    69→CALTEX MD is a full-featured WhatsApp multi-device bot built on the Baileys library. It provides a rich command system, multi-provider AI integration, a plugin architecture, group moderation tools, media handling, scheduled messaging, and a beautiful Next.js web dashboard for complete bot management.
    70→
    71→## Key Features
    72→
    73→### WhatsApp Integration
    74→- Multi-device support via Baileys MD
    75→- QR code authentication
    76→- **Pairing code authentication** — link via phone number without scanning
    77→- Auto-reconnect with configurable retry logic
    78→- Multi-session management
    79→- Message send/receive with full media support
    80→
    81→### AI Integration
    82→- Multi-provider support: **OpenAI**, **Gemini**, **Claude**, **Ollama**, **Custom**
    83→- Conversation memory with TTL-based cleanup
    84→- Chat, image generation, code generation, translation, summarization, rewriting, and explanation
    85→- Configurable system prompts and model parameters
    86→
    87→### Plugin System
    88→- Dynamic plugin loading and unloading
    89→- Custom command registration via plugins
    90→- Plugin enable/disable from the dashboard or WhatsApp commands
    91→- Hot-reload support for development
    92→- Install plugins from file paths
    93→- Full lifecycle management: install, remove, enable, disable, reload
    94→
    95→### Web Dashboard
    96→- Real-time bot status and health monitoring

    98→- **Pairing code panel** — link via phone number
    99→- Session management (create, delete, export, import)

   144→## Quick Start
   145→
   146→### Prerequisites
   147→
   148→- **Node.js** >= 20 or **Bun** >= 1.0
   149→- **npm** or **bun** package manager
   150→- A WhatsApp account with an active phone number
   151→- (Optional) AI provider API keys for AI features
   152→
   153→### Installation
   154→
   155→```bash
   156→# Clone the repository
   157→git clone https://github.com/caltex-md/caltex-md.git
   158→cd caltex-md
   159→
   160→# Install dependencies
   161→npm install
   162→# or with bun
   163→bun install
   164→
   165→# Set up environment variables
   166→cp .env.example .env
   167→# Edit .env with your configuration
   168→
   169→# Initialize the database
   170→npx prisma db push
   171→npx prisma generate
   172→
   173→# Build the dashboard
   174→npm run build
   175→```
   176→
   177→### Running the Bot
   178→
   179→```bash
   180→# Start all services (Dashboard + Bot + WebSocket)
   181→npm start
   182→
   183→# Development mode (Dashboard only)
   184→npm run dev
   185→
   186→# Start bot service separately
   187→cd mini-services/caltex-bot
   188→bun run dev
   189→```
   190→
   191→When the bot starts, it will display a QR code in the console. Scan it with WhatsApp:
   192→
   193→> **WhatsApp** > **Settings** > **Linked Devices** > **Link a Device**
   194→
   195→### Using Pairing Code (New in v1.1.0)
   196→
   197→Instead of scanning a QR code, you can link your bot using a phone number:
   198→
   199→1. Open the **Dashboard** > **Pairing Code** panel
   200→2. Enter your WhatsApp phone number (e.g., `6281234567890`)
   201→3. Click **Request Pairing Code**
   202→4. Enter the received code on your phone:
   203→   > **WhatsApp** > **Settings** > **Linked Devices** > **Link with Phone Number**
   204→
   205→You can also request a pairing code via the API:
   206→
   207→```bash
   208→curl -X POST http://localhost:3000/api/pairing-code \
   209→  -H "Authorization: Bearer <your-token>" \
   210→  -H "Content-Type: application/json" \
   211→  -d '{"phoneNumber":"6281234567890"}'
   212→```
   213→
   214→### Docker Quick Start
   215→
   216→```bash
   217→# Start all services with Docker Compose
   218→docker compose up -d
   219→
   220→# View logs
   221→docker compose logs -f
   222→
   223→# Stop all services
   224→docker compose down
   225→```

   229→## Configuration
   230→
   231→All configuration is managed through environment variables in the `.env` file:
   232→
   233→| Variable | Description | Default |
   234→|---|---|---|
   235→| `NODE_ENV` | Runtime environment | `development` |
   236→| `PORT` | Dashboard port | `3000` |
   237→| `DATABASE_URL` | SQLite database path | `file:./data/caltex.db` |
   238→| `BOT_API_URL` | Bot service URL | `http://localhost:3031` |
   239→| `WS_URL` | WebSocket service URL | `http://localhost:3003` |
   240→| `NEXTAUTH_SECRET` | NextAuth secret key | `change-me-in-production` |
   241→| `NEXTAUTH_URL` | NextAuth callback URL | `http://localhost:3000` |
   242→| `JWT_SECRET` | JWT signing secret | `caltex-md-bot-secret-key-2024` |
   243→| `ADMIN_USER` | Dashboard admin username | `admin` |
   244→| `ADMIN_PASS` | Dashboard admin password | `admin123` |
   245→| `PASSWORD_SALT` | Password hashing salt | `caltex-salt` |
   246→
   247→Bot-specific configuration is also managed via the `bot-config.json` and `ai-config.json` files, which can be edited from the dashboard.

   740→## Plugin Development
   741→
   742→CALTEX MD supports a plugin architecture for extending bot functionality.
   743→
   744→### Plugin Structure
   745→
   746→```typescript
   747→import type { Plugin, CommandContext } from '@/lib/plugin-loader';
   748→
   749→const myPlugin: Plugin = {
   750→  name: 'my-plugin',
   751→  version: '1.0.0',
   752→  description: 'My custom plugin',
   753→  author: 'Your Name',
   754→  enabled: true,
   755→  commands: [
   756→    {
   757→      name: 'hello',
   758→      description: 'Say hello',
   759→      category: 'custom',
   760→      aliases: ['hi', 'hey'],
   761→      cooldown: 3,
   762→      isPremiumOnly: false,
   763→      handler: async (ctx: CommandContext) => {
   764→        await ctx.reply('Hello from my plugin! 👋');
   765→      },
   766→    },
   767→  ],
   768→};
   769→
   770→export default myPlugin;
   771→```
   772→
   773→### Command Context
   774→
   775→Each command handler receives a `CommandContext` object with:
   776→
   777→| Property | Type | Description |
   778→|---|---|---|
   779→| `sock` | `WASocket` | WhatsApp socket instance |
   780→| `jid` | `string` | Chat JID |
   781→| `sender` | `string` | Sender JID |
   782→| `args` | `string[]` | Command arguments |
   783→| `text` | `string` | Full message text |
   784→| `isGroup` | `boolean` | Whether the message is from a group |
   785→| `isOwner` | `boolean` | Whether the sender is a bot owner |
   786→| `isAdmin` | `boolean` | Whether the sender is a group admin |
   787→| `reply(text)` | `function` | Reply to the message |
   788→| `react(emoji)` | `function` | React to the message |
   789→
   790→### Installing Plugins
   791→
   792→1. **Via WhatsApp command:** `!installplugin ./custom-plugins/my-plugin.ts`
   793→2. **Via dashboard:** Plugins panel > Install
   794→3. **Via file system:** Place your plugin file in `src/lib/commands/` and it will auto-load
   795→4. Enable/disable from the dashboard Plugins panel or via `!enableplugin`/`!disableplugin`
   796→5. Hot-reload during development with `!reloadplugin <name>`
